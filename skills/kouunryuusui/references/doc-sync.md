# SSOT宣言プロトコル

## 目的

各フェーズで「何を根拠に作業しているか」を明示する。
Jira・Notion・ローカルMDに同じ内容が散在する場合、どれが正かをちぃが判断できるようにする。

## SSOTの移行

| フェーズ | 根拠（SSOT） |
|---------|------------|
| E1開始前 | Jiraチケット `{ID}` |
| E1完了後 | `spec-draft.md` |
| E2完了後 | `design.md` |
| E3完了後 | `ticket-plan.md` |

## 宣言フォーマット

各フェーズ完了時に `scratch.md` に追記する:

```markdown
## SSOT宣言
- **根拠**: {ファイル名 or チケットID}
- **更新**: {ISO 8601}
- **変更前**: {前のSSOT}
```

## 判断ルール

複数の情報源（Jira/Notion/ローカルMD）で内容が異なる場合:
1. **現在フェーズのSSOTファイルを正とする**
2. 外部ツールの内容は参考にとどめる
3. 差異がある場合は scratch.md に記録する
