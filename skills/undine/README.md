# undine — 水先案内人

> 「どのスキルを使えばいいの？」に答える、senjuのナビゲーター。

ARIAの水先案内人（ウンディーネ）のように、ユーザーを迷わず目的のスキルへ案内する。

---

## 何をするスキルか

senjuには17のスキルがある。似たようなスキルが複数あったり、どれから始めればいいか分からない場面がある。  
`undine` はそんな時の入口。ユーザーの意図を聞き取り、最適なスキルを1〜3個提案し、
**そのまま sub-agent または team で実行まで行う**。

## 起動方法

### 明示起動

```
/undine
```

または:

- 「どのスキルを使えばいい？」
- 「何から始めれば？」
- 「スキルを案内して」
- 「おすすめは？」

### 自動起動

スキルを明示せず曖昧な開発タスクを依頼した場合に自動発火する。  
ただし、**別のスキルが実行中の場合は発火しない**。

---

## 出力例

```
## おすすめスキル

**あなたの目的**: 大きなEpicを設計から実装・PRまで一気にやりたい

---

### 第一候補: `kouunryuusui`
**なぜ？**: Epic/US → AC生成 → 設計 → チケット分割 → 実装 → PR まで
           自律実行する統合フロー。これがまさに目的に合致する。
**起動方法**: `/kouunryuusui`
**向いている状況**: 要件がある程度固まっていて、設計〜実装まで一気に進めたい時

### 第二候補: `vertical-slice`
**なぜ？**: Walking Skeleton で最小動作品を最初に作り、段階的に拡張する。
           kouunryuusui の中でも内部的に使われる設計戦略。
**起動方法**: `/vertical-slice`
**向いている状況**: 機能の全体像を掴みながら段階的に実装したい時

---

### 組み合わせパターン
`vertical-slice` で設計戦略を決めてから `kouunryuusui` で実装 →
段階的な設計と品質保証が両立できる。
```

---

## 提案〜実行ロジック

1. **意図の把握** — 不明なら選択肢形式で質問
2. **カタログの動的収集** — `skills/*/SKILL.md` の description を実際に読む（ハードコードしない）
3. **マッチング** — キーワード・目的・開発フェーズで照合
4. **提案** — 上位1〜3候補 + 根拠 + 組み合わせパターン
5. **起動** — ユーザーが選んだらそのスキルを sub-agent（executor / deep-executor）または team で実行

---

## 関連スキル

undine はすべてのスキルへの入口になるため、特定スキルとの依存関係はない。  
提案先のスキルを参照のこと。
